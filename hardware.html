Hardware and headsets

What the work actually runs on. Every category before this one defaults to a handheld phone; this page is about the cases where a headset changes what an installation can ask of a visitor.

Meta Quest (passthrough MR) Standalone headset with built-in hand tracking and passthrough video, so the real room and virtual content blend without a phone in hand. One-time hardware cost, no subscription for the device itself Medium to high, usually via Unity or native SDKs; WebXR-in-headset-browser is a lighter option App store (Meta Horizon Store) or sideload Yes, once installed Medium. WebXR-in-headset stays vibecodable; the native SDK path inherits the same constraints as the native and engine category.
Apple Vision Pro visionOS headset, RealityKit and SwiftUI, the highest passthrough fidelity available right now. Expensive hardware, plus a Mac and Apple Developer account for development High App Store or ad-hoc install Yes, once installed Medium, same profile as ARKit in the native and engine category, with a much smaller installed base to reach.
HoloLens 2 An enterprise-focused optical see-through headset (a genuinely transparent display, not a video passthrough). Microsoft has deprioritized new hardware here, so treat this row as legacy rather than a current recommendation. Expensive, enterprise-oriented High Enterprise deployment tools, not a consumer app store Yes, once installed Low. Niche tooling and a shrinking ecosystem.
Smart glasses Two different things worth telling apart: Snap Spectacles (a real, if limited, AR display with a Lens Studio development path) and Ray-Ban Meta glasses (camera, audio, and on-device AI, with no AR display at all as of this writing). Developer kits and consumer units vary widely Depends on the device; Spectacles inherits Lens Studio's curve Platform-locked in both cases No Low. Same constraint as the no-code platforms these devices are tied to.
None of the projects behind this atlas run on dedicated hardware yet. Everything so far targets a handheld phone browser, by design: it is the option with zero install friction for gallery visitors and workshop participants. A headset would only make sense for a piece that specifically needs both hands free for an extended, multi-visitor gallery run.
